# IR Model

The intermediate representation lives in `BindingPackage`.

## BindingItem variants

| Variant | Description |
|---------|-------------|
| `Function` | C function declaration or definition |
| `Record` | struct or union (may be opaque) |
| `Enum` | enumeration with named variants |
| `TypeAlias` | typedef |
| `Variable` | global variable declaration |
| `Unsupported` | construct that could not be fully represented |

## BindingType

Primitive types: `Void`, `Bool`, `Char`, `SChar`, `UChar`, `Short`, `UShort`, `Int`, `UInt`, `Long`, `ULong`, `LongLong`, `ULongLong`, `Float`, `Double`, `LongDouble`.

Compound types: `Pointer`, `Array`, `FunctionPointer`, `TypedefRef`, `RecordRef`, `EnumRef`, `Opaque`.

## Serialization

The entire IR is serde-compatible:

```rust
let json = bic::to_json(&result.package).unwrap();
let pkg = bic::from_json(&json).unwrap();
```
